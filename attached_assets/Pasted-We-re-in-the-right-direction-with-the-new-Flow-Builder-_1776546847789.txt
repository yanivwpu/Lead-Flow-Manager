We’re in the right direction with the new Flow Builder, but now we need a **v2 polish pass** to make it feel more premium, more intentional, and more trustworthy. The current redesign is already much better than the old Chatbot Builder, but it still feels like an in-between state: cleaner, but not yet fully polished or “wow.”

Please do a focused polish pass on the current Flow Builder UI and behavior.

## Overall goal

Push the current Flow Builder from “good redesign” to “premium product feel.”

This pass should focus on:

* stronger visual hierarchy
* better clarity
* more confident system feedback
* slightly more personality and guidance
* fixing broken / unclear actions like Duplicate

Do not redesign from scratch again. Build on the current Flow Builder structure.

---

## 1. Fix the Duplicate issue first

Current problem:

* The overflow menu shows **Duplicate**
* It either does nothing or gives no visible result
* This makes the product feel broken and lowers trust immediately

### Required fix

Change **Duplicate** to **Duplicate Flow**

When clicked:

1. Create a full copy of the current flow
2. Copy all:

   * flow name
   * description
   * trigger settings
   * steps
   * step settings/configurations
   * step order
3. Name the new flow:

   * `{Original Name} (Copy)`
4. Set duplicated flow status to:

   * Draft / inactive
5. Automatically refresh the sidebar
6. Automatically select and open the duplicated flow
7. Show visible confirmation:

   * toast or inline success message:

     * **Flow duplicated**
8. If duplication fails:

   * show proper error message
   * do not fail silently

### Important

If duplication cannot be fully implemented right now, remove the Duplicate Flow option temporarily from the menu until it works properly.

Do not leave non-working actions in the UI.

## 3. Make Trigger feel more important

The Trigger card is correct in structure, but currently it still feels like just another form card.

Trigger is the most important logic block in the flow. It should feel like the starting brain of the automation.

### Please improve the Trigger block:

* Stronger title hierarchy
* Better supporting copy
* Slightly stronger visual emphasis than regular step cards
* Make it feel like the entry point of the flow

### Copy improvements

Replace:

* **Choose when this flow starts**

With something like:

* **This flow runs when these conditions are met**
  or
* **Set when this flow should start**

For the “Start on new conversation” description, make it slightly tighter and clearer.

For example:

* **Runs when a contact starts a new conversation**

### Trigger warning state

The warning is useful, but should be more polished.

Current idea is correct, but improve copy to something cleaner:

* **No trigger set. Add a keyword or enable “Start on new conversation” so this flow can run.**

The warning should feel helpful, not alarming or clunky.

## 4. Improve step card hierarchy

The step cards are heading in the right direction, but still feel a little generic and too soft.

We need clearer information hierarchy so a user can scan the flow faster.

### Current problem

Right now the structure often reads like:

* Welcome Message
* Send Message
* Hello! How can I help you today?

But the most important thing is often the **action type**, not the custom title.

### Improve card hierarchy so it is easier to scan

Examples:

* Action type should stand out more
* Step name should still exist, but not compete awkwardly
* Preview text should be easier to read at a glance

### Goal

A user should quickly understand:

* what type of step it is
* what it does
* what comes next

### Suggested direction

For example, a message step should visually read more like:

* **Send Message**
* Welcome Message
* Hello! How can I help you today?

Or another clean hierarchy that makes action type more obvious than today.

Apply similar logic to:

* Ask Question
* Wait
* Action
